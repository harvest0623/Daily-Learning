## 万条数据渲染
> **渲染引擎的耗时很大**

### 1. 一次性渲染：白屏 + 闪屏
- 渲染所有数据，会导致页面卡顿
- 闪屏：在渲染完成后，页面会显示一个空白的区域，然后才会显示实际的内容
### 2. 分页渲染：白屏 + 分页渲染耗时
- setTimeout 定时器的时间不是确定的，可能会有误差
- requestAnimationFrame 定时器的时间是确定的
### 3. 虚拟列表
- 只渲染当前可见的区域，其他区域不渲染
- 过程：
    - 计算当前可见的区域
    - 渲染当前可见的区域
    - 监听滚动事件，当滚动事件触发时，重新计算当前可见的区域
    - 渲染当前可见的区域
