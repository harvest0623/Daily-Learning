## 构建工具
### 1. webpack
- 介绍：一个静态模块打包器，将模块打包成静态资源。
- 配置：webpack.config.js
- 只能打包js文件
- 能外挂插件
- 深度优先，递归式的做依赖收集，每次打包时，会根据入口文件，递归式的收集所有依赖的文件，直到没有依赖的文件为止，最后将所有收集到的文件打包成一个文件（构建比较耗时）

### 2. vite
- 介绍：一个基于事件流的构建工具，用于自动化重复性任务。
- 配置：vite.config.js
- 配置文件简单，插件丰富
- 支持异步任务

### 3. grunt
- 介绍：一个基于事件流的构建工具，用于自动化重复性任务。
- 配置：Gruntfile.js
- 优点：简单易用，插件丰富
- 缺点：配置文件复杂，不支持异步任务

### 4. gulp
- 介绍：一个基于事件流的构建工具，用于自动化重复性任务。
- 配置：gulpfile.js
- 优点：简单易用，插件丰富
- 缺点：配置文件复杂，不支持异步任务
