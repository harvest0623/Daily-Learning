# 三、谈谈 JS 中的类型转换机制？
1. **是什么：**
    - 显式类型转换：人为地将一个类型转换为另一个类型
    - 隐式类型转换：JS 引擎在执行运算或者判断语句等情况时，将两种不同类型的值转换为相同类型
2. **特点：**
    - 显式类型转换：使用 Number()、String()、Boolean() 等构造函数，或者调用对象的 toString()、valueOf() 方法
    - 隐式类型转换：四则运算符(+、-、*、/)，判断语句(if、while、for)，逻辑运算符(&&、||、!)，==、!=

# 四、== 和 === 有什么区别？
- ==：会进行类型转换，判断值是否相等
- ===：不会进行类型转换，判断值和类型是否都相等

# 五、聊一聊 JS 中的拷贝
1. **是什么：** 拷贝是指将一个对象的属性和方法复制到另一个对象中，实现对象的独立存在和操作
2. **分类：**
    - 基础类型：直接赋值，不会创建新的对象
    - 引用类型：创建新的对象，但是对象的属性和方法是共享的
3. **方法：**
    - 浅拷贝：只拷贝对象的引用，而不是对象本身
    - 深拷贝：拷贝对象及其所有子对象，实现对象的独立存在和操作

> JSON.parse(JSON.stringify(obj)) 注意： 不能拷贝函数，不能拷贝 undefined，不能处理 symbol, bigint 类型的数据
>
> structuredClone(obj) 注意： 不能处理 symbol, 函数

4. **应用场景：**
    - 深拷贝：在需要独立存在和操作对象的情况下，如事件处理函数、定时器函数等
    - 浅拷贝：在需要共享对象的情况下，如数组、对象等

# 六、聊一聊 JS 中的闭包
1. **是什么：** 根据作用域的查找规则，内部函数可以访问外部函数的变量。一个函数执行完毕后，它的上下文会被销毁；那么，当一个函数 A 内部存在一个函数 B，而函数 B 被拿到 A 的外部调用，那么 A 函数中被 B 访问的那些变量就**不会被销毁，而是以一个集合的形式被保留在了调用栈中**，这个集合就叫做闭包
2. **作用：**
    - 闭包可以实现数据的封装和隐藏
    - 闭包可以实现函数的复用和共享
3. **应用场景：**
    - 模块化
    - 事件处理
    - 私有变量
    - 防抖和节流

# 七、说说你对 JS 作用域的理解
1. **是什么：** JS 作用域是指在代码中变量和函数的可见性和访问范围
2. **作用域分类：**
    - 全局作用域：在代码的最外部定义的变量和函数，整个代码都可以都可以访问
    - 函数作用域：在函数内部定义的变量和函数，只能在函数内部访问
    - 块级作用域：在块级代码块（如 if、for、while 等）内部定义的变量和函数，只能在块级代码块内部访问
3. **特点：**
    - 变量提升：在函数作用域中，变量和函数的声明会提升到函数的顶部，但是赋值不会提升到顶部
    - 内部作用域可以访问外部作用域的变量和函数，但是外部作用域不能访问内部作用域的变量和函数
4. **作用域链：**
    - 每个执行上下文都有一个作用域链，用于查找变量和函数
    - 作用域链由当前执行上下文的变量对象、父执行上下文的变量对象、全局变量对象等组成

# 八、聊聊 JS 的原型
1. **是什么：** 
    - 函数有二义性，既可以作为函数调用，也可以作为构造函数调用
    - 函数上有一个 prototype 属性，被 new 关键字调用时，会将原型对象上的属性和方法挂载到实例对象的隐式原型上，实例对象可以通过__proto__属性访问原型对象
2. **特点：**
    - 每个对象都有一个原型对象
    - 原型对象本身也是一个对象
    - 原型对象的原型对象是 null
3. **应用场景：**
    - 当需要创建多个实例对象时，可以将公共的属性和方法挂载到原型对象上，每个实例对象都可以访问到这些属性和方法
4. **Object.create(null)：**
    - 创建一个空对象，没有原型对象

# 九、说一下 JS 中的继承
1. **是什么：** 继承是指一个对象可以继承另一个对象的属性和方法
2. **分类：**
    - **原型链继承：** 子类之间共用了同一个原型对象，会相互影响
    - **构造函数继承：** 子类可以继承到父类的属性，但是不能继承到父类原型上的属性
    - **寄生组合继承：** 通过将原型链继承和构造函数继承结合起来，可以避免原型链继承的问题
    - **class 继承：** 通过 extends 关键字实现继承，以及 super 关键字调用父类的构造函数

# 十、说说你对 JS 中 this 的理解
1. **是什么：** this 是函数在运行时自动生成的一个内部对象，this 代指谁取决于函数的调用方式
2. **特点：**
    - 函数独立调用，this 指向 window
    - 函数被对象调用，this 指向该对象
    - 函数被多层对象调用，this 指向最近的对象
    - 显示绑定：call apply bind
    - new 绑定: this 指向新创建的实例对象
    - 箭头函数：this 指向定义时的上下文对象，而不是调用时的上下文对象

# 十一、聊一下 JS 中的事件模型
1. **是什么：** 事件模型是指在 JS 中，事件的触发和处理机制
2. **分类：**
    - DOM2 事件模型
        - 事件冒泡：事件从目标元素开始，逐级向上传递，直到 window 元素
        - 事件捕获：事件从 window 元素开始，逐级向下传递，直到目标元素
        - 事件委托：将事件处理函数挂载到父元素上，而不是每个子元素上，实现事件的批量处理
    - DOM0 事件模型
        - onclick 事件：在元素上挂载 onclick 事件处理函数，当元素被点击时，会触发用该函数处理
        - onmouseover 事件：在元素上挂载 onmouseover 事件处理函数，当鼠标悬停在元素上时，会触发用该函数处理
        - onmouseout 事件：在元素上挂载 onmouseout 事件处理函数，当鼠标移出元素时，会触发用该函数处理
3. **特点：**
    - 事件冒泡和捕获可以同时存在
    - 事件冒泡和捕获可以被中断
    - 事件冒泡和捕获可以被阻止
4. **应用场景：**
    - 事件处理
    - 事件委托
    - 事件冒泡和捕获的使用

# 十三、说说你对 JS 事件循环机制的理解
1. **是什么：** 事件循环机制是指 JS 引擎在执行代码时，如何处理事件和异步操作
2. **分类：**
    - **事件循环：** 事件循环是指 JS 引擎在执行代码时，如何处理事件和异步操作
    - **事件循环队列：** 事件循环队列是指 JS 引擎在执行代码时，如何处理事件和异步操作的队列
3. **特点：**
    - 事件循环队列中可以同时存在多个事件
    - 事件循环队列中的事件可以被中断
    - 事件循环队列中的事件可以被阻止
4. **应用场景：**
    - 事件循环队列可以实现异步操作的处理
    - 事件循环队列可以实现事件的处理

# 十四、说说你对柯里化的理解
1. **是什么：** 柯里化是指将一个多参数函数转换为一系列单参数函数，每个函数只处理一个参数
2. **特点：**
    - 实现函数的复用和共享
    - 提高代码的适用性

# 十五、JS 中如何实现函数缓存
1. **是什么：** 函数缓存是指将函数的计算结果缓存起来，当下次调用函数时，如果参数相同，就直接返回缓存的结果，而不是重新计算
2. **特点：**
    - 提高函数的执行效率
    - 避免重复计算
    - 缓存结果可以被共享
3. **应用场景：**
    - 对于昂贵的函数调用，如网络请求、数据库查询等
    - 对于重复计算的函数调用，如计算阶乘、组合数等
    - 对于需要缓存结果的函数调用，如计算缓存、查询缓存等

# 十六、说说 JS 中数字精度丢失的问题
1. **是什么：** 
    - 数字精度丢失是指在 JS 中，数字的表示范围和精度有限，超过范围或精度的数字会被四舍五入，导致精度丢失
    - **浮点数：** 是一种数字的标准，Number 采用 IEEE 754 浮点数表示法，64 位双精度存储，其中 1 位符号位，52 位尾数，11 位指数位

> X = a * 2^b
>
> 27.0 == 1 * 2^4 + 1 * 2^3 + 0 * 2^2 + 1 * 2^1 + 1 * 2^0

2. 解决方法：
    - **使用 Number.toFixed() 方法：** 将数字转换为固定小数位数的字符串，避免精度丢失
    - **使用 Number.round() 方法：** 将数字四舍五入，避免精度丢失
