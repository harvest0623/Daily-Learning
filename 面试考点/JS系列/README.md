# 三、谈谈 JS 中的类型转换机制？
1. **是什么：**
    - 显式类型转换：人为地将一个类型转换为另一个类型
    - 隐式类型转换：JS 引擎在执行运算或者判断语句等情况时，将两种不同类型的值转换为相同类型
2. **特点：**
    - 显式类型转换：使用 Number()、String()、Boolean() 等构造函数，或者调用对象的 toString()、valueOf() 方法
    - 隐式类型转换：四则运算符(+、-、*、/)，判断语句(if、while、for)，逻辑运算符(&&、||、!)，==、!=

# 四、== 和 === 有什么区别？
- ==：会进行类型转换，判断值是否相等
- ===：不会进行类型转换，判断值和类型是否都相等

# 五、聊一聊 JS 中的拷贝
1. **浅拷贝：**
    - 只拷贝对象的引用，而不是对象本身
    - 可以使用 Object.assign()、展开运算符(...) 实现
2. **深拷贝：**
    - 拷贝对象及其所有子对象
    - 可以使用 JSON.parse(JSON.stringify(obj)) 实现

# 六、聊一聊 JS 中的闭包
1. **是什么：** 闭包是指在函数内部定义的函数，该函数可以访问外部函数的变量
2. **作用：**
    - 闭包可以实现数据的封装和隐藏
    - 闭包可以实现函数的复用和共享
3. **应用场景：**
    - 模块化
    - 事件处理
    - 私有变量
    - 防抖和节流

# 七、说说你对 JS 作用域的理解
1. **是什么：** JS 作用域是指在代码中变量和函数的可见性和访问范围
2. **作用域分类：**
    - 全局作用域：在代码的最外部定义的变量和函数，整个代码都可以都可以访问
    - 函数作用域：在函数内部定义的变量和函数，只能在函数内部访问
    - 块级作用域：在块级代码块（如 if、for、while 等）内部定义的变量和函数，只能在块级代码块内部访问
3. **特点：**
    - 变量提升：在函数作用域中，变量和函数的声明会提升到函数的顶部，但是赋值不会提升到顶部
    - 内部作用域可以访问外部作用域的变量和函数，但是外部作用域不能访问内部作用域的变量和函数
4. **作用域链：**
    - 每个执行上下文都有一个作用域链，用于查找变量和函数
    - 作用域链由当前执行上下文的变量对象、父执行上下文的变量对象、全局变量对象等组成

# 八、聊聊 JS 的原型
1. **是什么：** 
    - 函数有二义性，既可以作为函数调用，也可以作为构造函数调用
    - 函数上有一个 prototype 属性，被 new 关键字调用时，会将原型对象上的属性和方法挂载到实例对象的隐式原型上，实例对象可以通过__proto__属性访问原型对象
2. **特点：**
    - 每个对象都有一个原型对象
    - 原型对象本身也是一个对象
    - 原型对象的原型对象是 null
3. **应用场景：**
    - 原型可以实现对象的继承
    - 原型可以实现对象的共享属性和方法
