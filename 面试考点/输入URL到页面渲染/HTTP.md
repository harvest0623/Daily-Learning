## HTTP 协议
1. HTTP 协议是一种**基于 TCP 协议的层协议**，用来在互联网上传输数据
2. HTTP 协议定义了客户端和服务器端之间的通信规则
3. HTTP 协议包括请求和响应两种消息
4. HTTP 协议使用文本格式进行通信，**浏览器和服务器之间通信语言是文本格式**

## HTTP/0.9 版本
1. HTTP/0.9 版本是一种简单的 HTTP 协议，用来传输体积个很小的 html 文件
2. 超文本传输协议
3. 用来传输体积个很小的 html 文件
4. 特点：
   - 只有请求行，没有请求头、请求体
   - 服务端没有响应头、响应体
   - 返回的内容是以 ASCII 码的字符流进行传输

## HTTP/1.0 版本
1. 网景公司推出浏览器，万维网联盟（W3C）, HTTP工作组成立
2. 网络发展带来了新的资源文件的传输需求 （图片、视频、音频、js、 css 等）
3. 如何实现多种类型的文件的下载？
   - 引入了请求头和响应头 （方便向服务器传递额外的信息） `accept: text/html` `accept-encoding: gzip, deflate` `accept-Charset: utf-8` `accept-language: zh-CN,zh;q=0.9` `content-encoding: br` `content-type: text/html; charset=utf-8`
   - 状态码
   - Cache机制
   - 用户代理字段

## HTTP/1.1 版本
1. 持久连接： 一次 TCP 连接可以传输多个请求和响应
2. 支持 6 个 TCP 持久链接
3. 队头阻塞问题
4. host 字段： 用来指定请求的目标主机
5. Chunk transfer encoding： 用来分块传输数据

## HTTP/2.0 版本
1. 1.1的问题 100 * n * RTT (100 * n * RTT) / 6 带宽利用率低 （100M - 12.5M）
2. 带宽利用率低：
   - 同时启用多条 TCP 连接，这写连接相互竞争带宽
   - TCP 的慢启动： 新建立的 TCP 连接，发送数据前，会先进行慢启动， 以避免对网络的拥塞
   - 队头阻塞问题： 当一个 TCP 连接中，某个请求的响应延迟了， 其他请求的响应也会被阻塞
3. 解决队头阻塞问题：
   - 只保留一个 TCP 连接， 其他请求都在这个连接上发送
   - 将该TCP 连接中的多个请求切片：每个请求片段还可以打上加急标签， 服务器可以根据标签，优先处理加急的请求
4. 头部压缩： 对请求头和响应头进行压缩， 减少了数据的传输量

## HTTPS === HTTP + SSL/TLS
1. TLS 是一种加密协议， 用来保护 HTTP 通信的安全
2. 对称加密：客户端和服务端都知道一个密钥， 用这个密钥加密数据， 也用这个密钥解密数据。
3. 非对称加密：客户端创建一个密钥，服务端创建一个公钥和一个私钥，服务端将公钥发送给客户端，客户端用公钥加密 密钥，传输给服务端，服务端用私钥解密密钥。
