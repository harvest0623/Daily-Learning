## 单页应用
### 1. 优点
- 组件化，提高代码的复用性
- 页面切换无刷新，用户体验好

### 2. 缺点
- 首次加载时间长，用户体验差
- 对搜索引擎不友好，SEO 优化难度大

## SSR 服务端渲染（Server-Side Rendering）
**将单页应用项目中的主页作为一个普通的 HTML 直接返回给浏览器，后续的页面切换由前端路由来处理**

## 强缓存
- 后端设置响应头 `Cache-Control: max-age=86400`，即可实现强缓存，浏览器会将响应缓存到本地，后续请求直接从缓存中读取，无需再请求服务器
- 浏览器 URL 地址栏的请求，不会触发强缓存
- 强缓存没有失效，后端更新了文件资源想让前端立马获取最新资源，文件名携带 hash 值

## 协商缓存
- 强缓存过期了，浏览器会发送请求到服务器，服务器会根据请求头中的 `If-Modified-Since` 时间戳判断资源是否有更新
- 如果资源没有更新，服务器会返回 `304 Not Modified` 响应，从而让浏览器会继续从缓存中读取资源
- 如果资源有更新，服务器会返回 `200 OK` 响应，浏览器会从响应中读取资源，同时更新缓存
