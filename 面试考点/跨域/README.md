## 一、同源策略
### 1. 是什么：浏览器自带的一种安全机制，限制不同源的资源访问
### 2. 作用：防止恶意网站获取用户信息
### 3. 实现：
- 协议：https://
- 域名：www.baidu.com
- 端口：80
- 路径：/home

## 二、跨域的解决方案
### 1. jsonp（jsonp with padding）
- **是什么：** 利用 script 标签的 src 属性可以跨域资源的特性，通过动态创建 script 标签向后端发送请求
- **缺点：** 
  - 只能发送GET请求，不支持POST等其他请求方法
  - 要求后端配合，返回数据时需要在回调函数名中添加参数
- **实现：**
  - 服务器端返回 json 数据，客户端解析数据
  - 客户端在请求参数中添加回调函数名，服务器端在返回数据中调用该函数，将数据作为参数传递

### 2. cors（cross origin resource sharing）
- **是什么：** 后端设置响应头，允许指定源访问资源，实现跨域请求
- **实现：**
  - 服务器端在响应头中添加 Access-Control-Allow-Origin 字段，指定允许的源
  - 客户端在请求头中添加 Access-Control-Allow-Methods 字段，指定允许的请求方法
  - 客户端在请求头中添加 Access-Control-Allow-Headers 字段，指定允许的请求头
  - 客户端在请求头中添加 Content-Type 字段，指定请求体的类型

### 3. proxy（node代理）
- **是什么：** 利用 node.js 的 http 模块，创建一个代理服务器，将请求转发到目标服务器，实现跨域请求
- **实现：**
  - 服务器端设置代理，将请求转发到目标服务器
